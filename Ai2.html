Arduino

So, I had this cool idea for a small Arduino-powered robot. The main feature was a tiny screen on the front that displayed different emojis to represent the robot's "emotions," and there was an LED light underneath that added a bit of flair. The twist was that it had obstacle avoidance—if you tried to block it, the robot would cleverly maneuver in the opposite direction and continue its path.

Project Overviews

Project Overviews

Arduino Project

To bring this concept to life, I started coding the Arduino. In the code, I defined the behavior for the emojis based on the robot's sensors. For obstacle avoidance, I integrated logic that utilized infrared or ultrasonic sensors to detect obstacles. If an obstacle was detected, the robot would dynamically display an "avoidance" emoji and swiftly change direction. The LED light would also change colors to match the current emotional state. I ensured the code was efficient and responsive to provide a delightful user experience.

‍

‍

Once the initial code was ready, it was time for testing. I carefully observed how the robot reacted to obstacles, making adjustments to sensor thresholds and reaction times. This phase involved a bit of trial and error—adjusting the speed, fine-tuning the obstacle detection, and refining the emoji display. LEDs were programmed to smoothly transition between colors to convey different moods. Testing was crucial to ensure the robot's movements were both interactive and entertaining.

‍

‍

With the code polished and the hardware components assembled, I uploaded the final program to the Arduino and let the robot loose. It roamed around, changing emojis based on its "emotions," and the LED light added a captivating visual element. When someone attempted to block its path, the robot skillfully maneuvered away, surprising and amusing everyone around. The implementation showcased how coding and creativity could blend seamlessly, turning a simple robot into an expressive and engaging companion.
